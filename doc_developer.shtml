Design Documentation

The design documentation is technical documentation, specifications or other information, typically for a specific, isolated feature. It is intended for software developers, and generally not understandable without a technical background of Equalizer. They are often moving targets, and the actual implementation may diverge from the original spec over time.

  • codingStyle.png illustrates the C++ coding style used in Equalizer, to ensure a consistent appearance for better readability.
  • threads.txt is a glossary and description of the individual threads in Equalizer.
  • Compounds is the initial design documentation on the API and config file interface for the configuration of scalable rendering.
  • Packet Handling explains the basic concept and mechanism of the Equalizer networking layer.
