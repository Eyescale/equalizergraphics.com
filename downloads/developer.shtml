Developer releases are regular, minor versions in-between major releases. Typically we tag a developer release after a feature has been implemented and tested. This ensures a reasonable stability, while providing early access to new features.

We do not provide packages of developer releases. Developer releases are snapshots of the source code, not the full trunk as for major releases. They can be checked out from the subversion repository using:
svn co https://equalizer.svn.sourceforge.net/svnroot/equalizer/tags/release-X.Y.Z

  • Version 0.5.3, 04. July 2008
  • Version 0.5.2, 16. June 2008
  • Version 0.5.1, 16. May 2008
