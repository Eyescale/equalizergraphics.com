Developer Releases

Developer releases are regular, minor versions in-between major releases. Typically we tag a developer release after a feature has been implemented and tested. This ensures a reasonable stability, while providing early access to new features.

We do not provide packages of developer releases. Developer releases are snapshots of the source code, not the full trunk as for major releases. They can be checked out from the subversion repository using:
svn co https://equalizer.svn.sourceforge.net/svnroot/equalizer/tags/release-X.Y.Z

The following developer releases based on the last major release are available:

  • Version 0.5.2, 16. June 2008
  • Version 0.5.1, 16. May 2008

Subversion Repository

The subversion repository gives access to the latest source code, and should be used by developers modifying the Equalizer source code or interested in the latest features. The tree can be checked out using subversion:
svn co https://equalizer.svn.sourceforge.net/svnroot/equalizer/trunk/src. We try to keep the svn tree stable and encourage developers to use the svn tree.

Make sure you check out trunk/src only, not the top-level repository as explained on sourceforge. Otherwise you will get all previous releases, well over one gigabyte of data.

A local, read-only copy is available for browsing the source code. The repository has the following layout and size:

GLAsync

GLAsync is a small, independent C++ class delivering asynchronous OpenGL downloads in the spirit of the SGIX_async extension.

GLAsync is completely independent from Equalizer. This download is offered for historical reasons, and the usage of the new ARB_sync extension is encouraged.
